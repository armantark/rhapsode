Pre-implementation plan

Make Japanese readable as Japanese, not as one long string.

The focused pass keeps Gemini prep-only and contract-first: structured output will carry Japanese lexical tokens, token readings, glosses, and translations; the reading UI will render token-level ruby as the primary Japanese surface.

Before / After

Current
ふたつの星空 こぼれ落ちたふたつの星が

Whole-line ruby fallback, no lexical boundaries, line-level gloss fallback when tokens do not exist.

Target
空そらsky こぼれ落ちたこぼれおちたspilled down ふたつふたつtwo ののgenitive 星ほしstars

Gemini supplies token segments; the frontend renders token-level readings as furigana and glosses under each word.

Work split

B1

Backend: extend prep structured output with optional token suggestions and readings, then write token children only where the line has no authored tokens.

B2

Prompt: keep XML-tagged context, no few-shot examples, and add Japanese-specific guidance for lexical tokenization, hiragana readings, and concise learner glosses.

F1

Frontend: render Japanese token children as the main line surface, with per-token ruby and interlinear glosses instead of a duplicated full-line block.

Q1

Verification: targeted backend and frontend tests, contract/client regeneration if the default prep layers change, and browser smoke via PinchTab.
