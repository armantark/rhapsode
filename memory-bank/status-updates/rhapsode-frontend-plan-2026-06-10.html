Pre-implementation plan · frontend lane

Rhapsode scholarly arcade

A SvelteKit interface for the existing contract: edit multilingual passages, drill them aloud with reference audio, self-grade in four strokes, and watch mastery compound. The backend is untouched; the generated OpenAPI client is the only bridge.

Contract

Client generated, never inferred

Types come from contracts/openapi.json via openapi-typescript. A thin fetch wrapper stamps every mutation with a stable Idempotency-Key and surfaces 409s as "create a new revision" flows.

Library & editor

Passages as structured text

A library view plus an editor for sections, lines, chunks, and tokens with per-segment cues and configurable annotation layers. Practiced revisions are immutable; edits fork a new revision.
