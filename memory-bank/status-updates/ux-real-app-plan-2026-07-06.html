Burrs first, then the missing surfaces

Your steer: the actual UX, not packaging. Real apps are distinguishable mostly by the absence of burrs — so those go first (both launcher bugs QA hit have root causes in hand) — then the three missing surfaces: Settings, a first-run sample, and discoverable shortcuts.

Burrs · root causes found

Launcher fixes

  • Double-click disclosure: the manual-modes <details> has no open binding, so the async data load re-renders it shut right after the first click. Fix: bind:open to state. (Closes the pending follow-up chip.)
  • Mode selection resets: chosenModes/chosenKinds are plain state — persist the last manual selection in localStorage like the minutes chip already is.
  • Wrong hint on typed cards: "press Space to check" is untrue while focus is in the typed-recall textarea (Space types a space). The hint becomes mode-aware.

Missing surface

Settings page

The app currently configures itself through a repo-root .env and env vars — the single biggest "dev tool" tell. New /settings route (plus nav link) backed by the existing AppSetting API:

  • Gemini key — masked input stored as a setting; prep.py reads setting-first, env-fallback. The practice loop still never touches the LLM.
  • Scheduler status — retention target, and whether personal FSRS weights are active or defaults (read-only; the fit stays a script per plan).
  • Backup health — folder path + "last snapshot: N hours ago" from a small GET /system/status.
  • Sound & mic defaults — the existing localStorage toggles, surfaced in one place.

First minutes + discoverability

Sample passage & shortcuts overlay

Sample: the empty library's dead-end gains "Load a sample passage" — a client-side create of Iliad 1.1–5 with line segments through the normal API, so the first five minutes teach the practice loop instead of demanding data entry. No backend change.

Shortcuts: a global ? overlay (Esc closes, inputs ignored) listing what's currently invisible: Space to check, 1–4 to grade, ⌘Z to undo, M to drop cue marks in the aligner. Rendered once in the layout.
