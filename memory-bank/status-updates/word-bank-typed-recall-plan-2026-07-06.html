PRE-IMPLEMENTATION PLAN · NEW EXERCISE MODES

Word-bank reconstruction → typed recall

Per your ruling: word bank when a line is young, graduating to typed recall as it matures; first-letter sprint dropped. Both are deterministic, offline, and self-verified — the check is a visual comparison the learner judges, never an automated matcher (same instrument philosophy as grill D3). Both become built-in modes: enum + contract regen + prompt shapes + card renderers.

EARLY · LEARNING STAGE

word_bank — rebuild the line from its own pieces

The line's units (token children when present, else words) are dealt as shuffled chips; tap to place them in order, tap a placed chip to take it back. Reveal shows the true line (with ruby / annotation layers) directly beneath the arrangement. Trains serial order — the dominant failure mode in verse — with all items given, so it slots right after fading in difficulty.

LATE · REVIEW / DURABLE

typed_recall — type it, then eyeball the match

Lead-in shown as the cue; the learner types the line from memory, then reveals. The typed attempt stays on screen with the true text rendered immediately below in the same passage font — mismatches (accents, breathings, okurigana) pop out visually, no diff algorithm anywhere. Grades on the same Again/Hard/Good/Easy bar; grade bar stays locked until the check, like all recall modes.
